BEFORE (Race Condition):
┌─────────┐              ┌─────────┐              ┌──────────────┐
│ User A  │              │ User B  │              │   Database   │
└────┬────┘              └────┬────┘              └──────┬───────┘
     │                        │                          │
  T1 │──SELECT available────────────────────────────────>│  qty_sold=99
     │<─────"1 available"──────────────────────────────┤│
     │                        │                          │
  T2 │                        │──SELECT available────────>│  qty_sold=99
     │                        │<─"1 available"──────────┤│
     │                        │                          │
  T3 │──UPDATE sold=100──────────────────────────────────>│  qty_sold=100 ✓
     │                        │                          │
  T4 │                        │──UPDATE sold=101─────────>│  qty_sold=101 ✗
     │                        │                          │  OVERSOLD!

AFTER (With Locking):
┌─────────┐              ┌─────────┐              ┌──────────────┐
│ User A  │              │ User B  │              │   Database   │
└────┬────┘              └────┬────┘              └──────┬───────┘
     │                        │                          │
  T1 │──SELECT FOR UPDATE───────────────────────────────>│  🔒 Lock row
     │<─────"1 available"──────────────────────────────┤│  qty_sold=99
     │                        │                          │
  T2 │                        │──SELECT FOR UPDATE───────>│  ⏳ WAIT (blocked)
     │                        │                          │
  T3 │──UPDATE sold=100──────────────────────────────────>│  qty_sold=100 ✓
     │<──────COMMIT───────────────────────────────────────│  🔓 Unlock
     │                        │                          │
  T4 │                        │<─"0 available"───────────│  qty_sold=100
     │                        │──ROLLBACK────────────────>│  ✓ Prevented!

╚═══════════════════════════════════════════════════════════════════════════╝

How It Works:
┌──────────────────────────────────────────────────────────────────────────┐
│  TicketClass:                                                            │
│    id: 1                                                                 │
│    quantity_sold: 99                                                     │
│    version: 5  ← Increments on every update                             │
└──────────────────────────────────────────────────────────────────────────┘

Transaction Flow:
┌─────────┐              ┌─────────┐              ┌──────────────┐
│ User A  │              │ User B  │              │   Database   │
└────┬────┘              └────┬────┘              └──────┬───────┘
     │                        │                          │
  T1 │──SELECT────────────────────────────────────────────>│  version=5
     │<─────Returns ticket with version=5────────────────┤│  sold=99
     │                        │                          │
  T2 │                        │──SELECT──────────────────>│  version=5
     │                        │<─Returns version=5───────┤│  sold=99
     │                        │                          │
  T3 │──UPDATE WHERE version=5────────────────────────────>│
     │   SET sold=100, version=6                        │  version=6 ✓
     │<──────1 row updated─────────────────────────────┤│  sold=100
     │                        │                          │
  T4 │                        │──UPDATE WHERE version=5──>│
     │                        │   SET sold=101, version=6│  version=6
     │                        │<─0 rows updated──────────┤│  CONFLICT!
     │                        │                          │
     │                        │  (Retry or show error)   │

SQL Generated:
  UPDATE ticket_classes 
  SET quantity_sold = quantity_sold + 1,
      version = version + 1
  WHERE id = 1 AND version = 5;  ← Only updates if version matches

        │                                             │
     T2 │──SELECT payments────────────────────────────>│  $1000
        │<─────Returns $1000──────────────────────────┤│
        │                                             │
     T3 │──SELECT refunds─────────────────────────────>│  $100
        │<─────Returns $100───────────────────────────┤│  (consistent)
        │                                             │
     T4 │──SELECT fees────────────────────────────────>│  $50
        │<─────Returns $50────────────────────────────┤│
        │                                             │
     T5 │──COMMIT TRANSACTION─────────────────────────>│  🔓 Release
        │                                             │
        │  Calculate: $1000 - $100 - $50 = $850       │  ✓ CORRECT!
